# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest release | :white_check_mark: |
| `main` branch | :white_check_mark: |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue, please report it responsibly.

### How to Report

**Please DO NOT file a public GitHub issue for security vulnerabilities.**

Instead, send an email to: **security@pruvon.dev**

Include the following information in your report:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes (optional)

### What to Expect

- **Acknowledgment**: We will acknowledge receipt of your report within 48 hours
- **Initial Assessment**: We will assess severity and impact
- **Updates**: We will keep you informed as we work on a fix
- **Disclosure**: We will coordinate public disclosure after a fix is available

## Security Best Practices

When using Pruvon:

- Keep your installation updated to the latest version
- Use strong passwords for admin accounts
- Protect your `config.yaml` file (contains sensitive credentials)
- Use HTTPS in production
- Regularly rotate API keys and secrets
- Follow the principle of least privilege for server access

## Scope

This security policy applies to:
- The main Pruvon application
- The web interface and API

Out of scope:
- Third-party services integrated with Pruvon
- User-created custom configurations
- Social engineering attacks
